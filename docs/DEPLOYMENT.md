# Deployment Guide

This document outlines the hosting architecture, CI/CD pipeline, and environment configuration required to host the **FlowStack ERP + CRM Operations Portal** live.

---

## 1. Hosting Architecture

The portal is separated into three highly optimized deployment segments:
1. **Frontend Client**: Vercel Static Hosting running the Vite + React client bundle.
2. **Backend API**: Railway Web Service running the Node.js + Express backend.
3. **Database**: Supabase hosting the PostgreSQL relational database.

Because both Vercel and Railway are linked directly to your GitHub repository, pushing to the `main` branch triggers an automated CI/CD pipeline that builds and deploys both services simultaneously.

---

## 2. Database Setup (Supabase)

Supabase provides the managed PostgreSQL database.

1. Create a new project on [Supabase](https://supabase.com/).
2. Navigate to **Project Settings -> Database** to find your Connection String (URI).
3. To initialize the database schema, run the following locally from your terminal, replacing `[YOUR_URI]` with your actual Supabase connection string:
   ```bash
   $env:DATABASE_URL="[YOUR_URI]"; $env:DATABASE_SSL="true"; npm --prefix backend run db:schema
   ```
4. To add the demo users (Admin, Sales, Warehouse, Accounts) to the live database, run:
   ```bash
   $env:DATABASE_URL="[YOUR_URI]"; $env:DATABASE_SSL="true"; npm --prefix backend run db:seed
   ```

## 3. Backend Deployment (Railway)

The backend handles the business logic, API routing, and raw PostgreSQL queries via `node-postgres`.

### Manual Setup via GitHub
1. Log in to [Railway](https://railway.app/).
2. Click **New Project** -> **Deploy from GitHub repo**.
3. Select the `flowstack-erp` repository.
4. Go to the project settings, and set the **Root Directory** to `/backend`.
5. Under **Networking**, ensure **Outbound IPv6** is enabled (required for Supabase direct connections).
6. Under **Variables**, add the following required environment variables:
   * `PORT` = `4000`
   * `DATABASE_URL` = `postgresql://postgres:[PASSWORD]@[HOST]:[PORT]/postgres`
   * `DATABASE_SSL` = `true`
   * `JWT_SECRET` = `your-secure-random-secret`
   * `JWT_EXPIRES_IN` = `8h`
   * `CORS_ORIGINS` = `https://flowstack-erp.vercel.app` (The frontend URL)
7. Railway will automatically detect the `package.json`, install dependencies, run the `build` script, and start the server using the `start` script.

---

## 4. Frontend Deployment (Vercel)

The frontend is a React Single Page Application bundled with Vite.

### Manual Setup via GitHub
1. Log in to [Vercel](https://vercel.com/).
2. Click **Add New...** -> **Project**.
3. Import the `flowstack-erp` repository from GitHub.
4. **CRITICAL**: Click "Edit" under **Root Directory** and select the `frontend` folder.
5. Vercel will automatically detect the framework as **Vite**.
6. Under **Environment Variables**, add:
   * `VITE_API_URL` = `https://flowstack-erp-production.up.railway.app` (The URL generated by Railway in the previous step)
7. Click **Deploy**.

---

## 5. Continuous Deployment (CI/CD)

Once set up, the deployment pipeline is entirely automated.

To update the live site:
1. Make code changes locally.
2. Commit and push your changes to GitHub:
   ```bash
   git add .
   git commit -m "update features"
   git push origin main
   ```
3. **Vercel** will intercept the webhook, rebuild the frontend bundle, and deploy the new UI.
4. **Railway** will intercept the webhook, rebuild the backend TypeScript code, and restart the Node.js server.
5. No manual intervention is required.
